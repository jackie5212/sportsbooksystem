# 小程序模拟登录说明

## 📝 修改内容

已将微信小程序的微信一键登录功能改为**模拟登录模式**,方便开发和测试,无需依赖后端服务。

## 🔧 修改位置

**文件**: `miniprogram/pages/login/login.js`

## 🎯 使用方式

### 开发环境(当前)
直接使用模拟登录,无需配置:
1. 打开微信开发者工具
2. 点击"微信授权登录"按钮
3. 自动使用模拟数据登录
4. 跳转到首页

### 切换到真实登录

当后端API准备好后,按以下步骤切换:

1. **打开文件**: `miniprogram/pages/login/login.js`

2. **注释掉模拟代码**:
   ```javascript
   // ========== 模拟登录开始 ==========
   // ... (将整个模拟登录部分注释掉)
   // ========== 模拟登录结束 ==========
   ```

3. **取消注释真实代码**:
   ```javascript
   /* ========== 真实登录代码(已注释) ========== */
   // 删除前后的 /* 和 */
   ```

4. **配置后端地址**:
   编辑 `miniprogram/app.js`:
   ```javascript
   globalData: {
     baseURL: 'https://your-domain.com/api' // 改为实际域名
   }
   ```

5. **测试登录**:
   - 清除缓存
   - 重新编译
   - 测试真实登录流程

## 💡 优势

### 模拟登录的优点:
1. ✅ **快速开发** - 无需等待后端完成
2. ✅ **独立测试** - 可以单独测试前端UI
3. ✅ **降低成本** - 不需要微信AppID
4. ✅ **灵活调试** - 可以随时修改模拟数据
5. ✅ **稳定可靠** - 不受网络和服务影响

### 何时切换到真实登录:
- 后端API已完成并部署
- 已申请微信小程序AppID
- 准备进行真机测试
- 准备提交审核发布

## 🔍 查看登录状态

登录成功后,可以在控制台查看:

```javascript
// 在微信开发者工具Console中
console.log(app.globalData.token)    // 查看Token
console.log(app.globalData.userInfo) // 查看用户信息
```

或者在Storage中看到:
- `token`: 模拟Token
- `userInfo`: 模拟用户数据

## ⚠️ 注意事项

1. **模拟数据的限制**:
   - OpenID是生成的,不是真实的
   - 手机号是固定的模拟值
   - 无法进行真实的订单操作

2. **切换时机**:
   - 建议在功能开发完成后切换
   - 真机测试前必须切换到真实登录
   - 提交审核前必须使用真实登录

3. **代码维护**:
   - 保留真实代码的注释,方便后续切换
   - 添加清晰的标记,区分模拟和真实代码
   - 定期更新模拟数据,保持合理性

## 📞 问题排查

### Q: 登录后看不到用户信息?
A: 检查 `app.js` 中的 `setUserInfo` 方法是否正确保存数据。

### Q: 如何清除模拟登录状态?
A: 
1. 在个人中心点击"退出登录"
2. 或手动清除Storage中的token和userInfo
3. 或重新编译小程序

### Q: 模拟登录会影响其他功能吗?
A: 不会。模拟登录只是生成Token和用户信息,其他功能正常使用。
